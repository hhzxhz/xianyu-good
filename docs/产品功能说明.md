# 闲鱼抢购后台 — 产品功能说明

## 一、产品概述

**闲鱼抢购后台（xianyu-good）** 是一款面向闲鱼 App 的自动化抢购与统计服务：支持多台 Android 手机接入，按关键词与规则自动搜索、监控新发/降价商品，对符合条件商品自动下单，并区分统计「自己抢到」与「被他人抢走」的结果。

- **适用场景**：抢购闲鱼新发/降价商品，多机多任务并行。
- **使用方式**：Web 管理后台 + REST API，需电脑连接手机（USB 调试 + uiautomator2）。

## 二、核心功能

### 2.1 设备管理

| 功能 | 说明 |
|------|------|
| 设备发现 | 通过 ADB 列出当前 USB 连接的 Android 设备（序列号、型号等）。 |
| 设备接入 | 将设备绑定到后台，支持备注名、启用/停用。 |
| 设备归属 | 设备可归属到用户；停用设备时自动停止该设备下所有运行中任务。 |
| 设备移除 | 删除设备时级联删除其下所有任务及记录。 |

### 2.2 抢购任务

| 功能 | 说明 |
|------|------|
| 任务创建 | 选择设备、填写搜索关键词、任务名称；可选最低价/最高价、描述关键字。 |
| 多规则 | 支持多条抢购规则（描述关键字 + 价格区间），规则间为「或」，满足任一条即抢。 |
| 运行模式 | **从 App 定位**：每次从闲鱼首页进入搜索 → 新发；**仅新发/降价**：直接在新发/降价 tab 循环刷新。 |
| 刷新间隔 | 可配置新发页列表刷新间隔（秒），控制轮询频率。 |
| 启停控制 | 任务可随时启动/停止；停止后轮询尽快退出，不重复下单。 |
| 任务列表 | 展示任务及规则、运行状态；普通用户仅看自己设备的任务，管理员看全部。 |

### 2.3 抢购流程（自动化）

1. **定位到搜索页**：从闲鱼首页 → 点击搜索区 → 填充关键词 → 键盘提交 → 进入搜索结果页 → 点击「新发」tab。
2. **新发/降价循环**：按任务刷新间隔在「新发 → 降价 → 新发」间切换并刷新（不点「综合」「价格」tab 解析商品）。
3. **列表解析**：解析当前页商品（标题/描述、价格）；仅在「新发」「降价」页解析，「价格」页不解析。
4. **规则匹配**：对每条商品按任务的规则判断（描述包含关键字 + 价格在区间内）；满足任一条规则即进入下单。
5. **自动下单**：点击商品进入详情 → 点击「立即购买/我想要」→ 记录结果（自己抢到 / 被他人抢走）。
6. **去重与节流**：同商品（描述+价格）不重复下单；3 分钟内对同一商品只尝试购买一次，避免重复点击。

### 2.4 统计与记录

| 功能 | 说明 |
|------|------|
| 汇总统计 | 按任务或全局汇总：自己抢到数量、被他人抢走数量、总记录数。 |
| 抢购记录 | 分页列表，可按任务、状态（自己抢到/被抢走）筛选；展示商品标题、价格、时间等。 |
| 解析商品 | 展示每轮解析到的新发/降价商品（去重保存），便于核对监控范围。 |
| 记录清理 | 支持按单条或批量删除抢购记录、解析记录。 |

### 2.5 运行与运维

| 功能 | 说明 |
|------|------|
| 任务控制台 | 每个任务有实时日志（控制台缓存）；仅在有运行中任务时按间隔轮询，减少请求。 |
| 运行中任务列表 | 接口返回当前正在执行的任务 ID 列表，便于前端展示状态。 |
| 启动恢复 | 服务重启后自动恢复所有「运行中」任务，无需手动重新启动。 |
| 多任务并发 | 每任务独立线程 + 独立事件循环，多机多任务互不阻塞。 |

## 三、用户与权限

| 功能 | 说明 |
|------|------|
| 验证码登录 | 支持手机号/邮箱 + 验证码登录。 |
| 密码登录 | 支持账号密码登录（如管理员）。 |
| 当前用户 | 获取/更新当前用户信息。 |
| 权限隔离 | 普通用户仅能查看、操作自己名下设备的任务与统计；管理员可管理所有用户与设备。 |

---

## 四、管理后台

| 功能 | 说明 |
|------|------|
| 后台入口 | 访问 `/admin`（建议禁用缓存以便及时看到更新）。 |
| 手机管理 | 查看 ADB 设备、接入/编辑/移除手机、启用停用。 |
| 任务管理 | 创建/编辑/删除任务，配置规则与刷新间隔，启动/停止任务。 |
| 抢购任务面板 | 展示任务列表、运行状态、控制台日志（有运行中任务时轮询）。 |
| 统计与记录 | 查看汇总、抢购记录、解析商品列表。 |
| 管理员 | 管理员可管理用户（列表、编辑）。 |

## 六、配置与部署

- **端口 / 数据库**：通过环境变量或 `.env` 配置（如 `PORT`、`DATABASE_URL`）。
- **轮询参数**：`poll_interval`（抢购轮询间隔）、`search_cooldown`（搜索后等待时间）等。
- **闲鱼包名**：在 `config` 中配置，用于 u2 控件定位。

## 七、功能一览表

| 模块 | 功能点 |
|------|--------|
| 设备 | 发现、接入、备注、启用/停用、移除、归属用户 |
| 任务 | 创建、多规则、运行模式、刷新间隔、启停、列表、控制台 |
| 抢购 | 首页→搜索→新发、新发/降价循环、列表解析、规则匹配、自动下单、去重节流 |
| 统计 | 汇总、抢购记录、解析商品、记录删除 |
| 用户 | 验证码/密码登录、当前用户、权限隔离 |
| 管理 | 后台页、手机/任务/统计管理、管理员用户管理 |

以上为当前产品的功能说明，技术细节与 API 可参见项目根目录 `README.md` 及接口文档 `/docs`。
